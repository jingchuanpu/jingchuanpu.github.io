XGBoost vs Random Forest — Play Tennis (14 days)
Sequential error correction vs parallel averaging

Both methods use multiple decision trees. The fundamental difference is how the trees relate to each other.

For the tennis dataset we encode Play=Yes as +1 and Play=No as −1. XGBoost starts with a single base prediction for all days, then builds shallow trees that each push the prediction closer to the true label by targeting what the current model gets wrong.

Click through the tabs to watch XGBoost build its ensemble round by round on all 14 tennis days.

Round 0 — base prediction (before any tree)

XGBoost starts with the simplest possible prediction for every instance: the log-odds of the training set. With 9 Yes (+1) and 5 No (−1):

base score = log(9/5) = log(1.8) ≈ 0.588
probability = sigmoid(0.588) ≈ 0.643 → predict Yes for all 14 days

This is already 9/14 correct (all the Yes days). But every No day is wrong. Those errors are the residuals Tree 1 must fix.

Round 1 — Tree 1 fits the residuals

Tree 1 is a shallow stump (depth=1) trained to predict the residual errors from Round 0. The best split is Outlook — it separates the large negative residuals (No days: D1,D2,D6,D8,D14) from the positive ones.

The learning rate η=0.3 means Tree 1 only contributes 30% of its full correction — this prevents overshooting. The remaining errors become the target for Tree 2.

Round 3 — Tree 3 makes fine corrections

By Round 3, the large errors from the No days are mostly handled. Tree 3 targets the smaller residuals — days that were close but not perfectly classified yet.

Notice how Tree 3's leaf values are much smaller than Tree 1's. This is the defining feature of boosting — early trees make large corrections on obvious errors, later trees make tiny fine-tuning adjustments. The corrections shrink naturally as residuals shrink.

F₃(x) = F₂(x) + η × Tree3(x)
       = base + η·T1 + η·T2 + η·T3

With more rounds (typical: 100–1000), predictions converge tightly to the true labels.

Final ensemble: 3-round XGBoost predictions

The final score for each day is the sum of the base prediction plus all 3 tree contributions. Positive scores → predict Yes; negative → predict No.

Final prediction = sign( base + Σ η·Treeₜ(x) )
Probability = sigmoid( base + Σ η·Treeₜ(x) )

Unlike Random Forest which outputs a hard vote, XGBoost outputs a continuous score that can be converted to a probability — making it more informative and better calibrated.

What it reduces
Variance — averaging de-correlated trees smooths out overfitting.
Bias — each round the ensemble gets closer to the true answer.

3 trees vote → majority wins. Clean, robust, no sequential dependency.

On a tiny 14-row dataset like tennis both methods achieve 100% training accuracy easily — the difference becomes apparent on larger, noisier, real-world data. XGBoost's sequential error correction gives it an edge in bias reduction, at the cost of being more sensitive to hyperparameters (especially learning rate and number of rounds).
