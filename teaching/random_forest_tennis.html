On average, ~63% of original rows appear in each bootstrap sample; the remaining ~37% are out-of-bag (OOB) and serve as a free validation set for that tree.

At each split, each tree only considers a random subset of √4 ≈ 2 features (feature subsampling). This forces variety — trees can't all pick Outlook first.

Tree 3
Features available at root: Outlook, Temp
Root: Outlook (best of Outlook, Temp)

Each tree sees different data AND different features at each split. Tree 2 never saw Outlook at the root — it had to rely on Humidity instead. This de-correlation is what makes averaging effective: errors don't all point the same way.

Given a new day to classify, each tree casts a vote. The majority wins. Let's try two new instances.

With more trees (e.g. 100 or 500), votes are rarely unanimous — the majority rule absorbs individual tree errors and gives a stable prediction.

Feature importance (Mean Decrease in Impurity) counts how much each feature reduces entropy across all splits in all trees, weighted by the number of samples reaching each split.

Each instance is left out of ~37% of trees. Those trees can predict it without having trained on it — giving a free, unbiased validation estimate called the OOB error.

Why OOB matters

No need for a separate validation set — you get validation for free from the training process itself.
Equivalent to leave-one-out cross-validation as the number of trees grows large.
Use it to tune hyperparameters (e.g. max_depth, n_estimators) without touching your test set.
